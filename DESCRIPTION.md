# PureMilk Quality Monitor — Описание функционала

## Общая информация

**PureMilk Quality Monitor** — веб-приложение для мониторинга качества сырого молока, поступающего от предприятий-поставщиков. Система позволяет хранить, анализировать и визуализировать данные лабораторных анализов каждой партии молока, автоматически рассчитывать сорт по настраиваемым нормативам и отслеживать историю аудитов хозяйств.

**Технологии:** React (фронтенд), FastAPI + PostgreSQL (бэкенд)
**Язык интерфейса:** Русский

---

## Навигация

В верхней панели расположены:
- **Логотип PureMilk** — переход на главную страницу
- **Поставки** — список партий молока
- **Предприятия** — справочник поставщиков
- **Аудиты** — журнал аудитов хозяйств
- **Аналитика** — сводные графики
- **Нормативы** — управление сортами и граничными значениями
- **Выпадающий список предприятий** (правый угол) — глобальный фильтр, применяется ко всем страницам

---

## 1. Страница «Поставки»

Главная рабочая страница системы. Отображает все партии молока в виде широкой таблицы с горизонтальной прокруткой.

### Таблица партий

Каждая строка = одна принятая партия молока. Колонки разбиты на тематические группы:

| Группа | Показатели |
|---|---|
| **Идентификация** | Дата, Предприятие, Вес (кг) |
| **Сорт** | Рассчитанный сорт (Экстра / Спец. I / Спец. II / Вне спец.) |
| **Температура** | ПО (°C), Т001-2 |
| **Органолептика** | ПО, Т001-2 |
| **Соматика** | Соматические клетки (тыс/мл) |
| **КМАФАнМ** | ПО, Т001-2 (тыс. КОЕ/мл) |
| **Т замерзания** | ПО, Т001-2 |
| **Состав** | Жир %, Белок %, Лактоза %, СОМО % |
| **Физ. свойства** | Плотность (кг/м³), Алкогольная проба %, Кислотность (°T) |
| **pH** | ПО, Т001-2 |
| **БГКП** | КОЕ/мл |
| **СЖК** | Свободные жирные кислоты |
| **Мочевина** | мг/100 мл |
| **Клостридии** | Споры клостридий (НВЧ/л) |
| **AB** | Наличие антибиотиков |

**Цветовая индикация** — показатели автоматически окрашиваются по нормативам:
- 🟢 Зелёный — норма
- 🟡 Жёлтый — приближается к границе
- 🔴 Красный — норматив нарушен

### Управление столбцами

Кнопка **⚙ Столбцы** (правый верхний угол) открывает панель выбора видимых колонок:
- Включение/выключение отдельных столбцов
- Включение/выключение целой группы одним чекбоксом
- Кнопки «Все» и «Мин.» для быстрого переключения

### Фильтры

Кнопка **☰ Фильтры** разворачивает панель на всю ширину страницы (сдвигает таблицу вниз). Панель содержит 18 групп фильтров:

- **Идентификация:** диапазон дат, вес (от/до)
- **Сортность:** выбор сорта (Экстра / Спец. I / Спец. II), наличие антибиотиков
- **Соматика:** SCC от/до
- **КМАФАнМ:** бактерии от/до
- **Температура:** от/до
- **Органолептика:** от/до
- **Точка замерзания:** от/до
- **Жир, Белок, Лактоза, СОМО:** каждый от/до
- **Плотность, Кислотность, Алк. проба:** от/до
- **pH:** от/до
- **БГКП, СЖК, Мочевина, Клостридии:** от/до

Кнопки **Применить** и **Сбросить всё**. Счётчик активных фильтров отображается на кнопке.

### Глобальный фильтр по предприятию

Выпадающий список в навигационной панели позволяет выбрать одно предприятие или «Все предприятия». Фильтр применяется немедленно и работает в связке с остальными фильтрами.

### Пагинация

25 записей на странице. Отображается диапазон «1–25 из N» и кнопки навигации.

## 2. Страница «Предприятия»

Справочник поставщиков молока.

### Список предприятий

Таблица с колонками: Предприятие (полное и краткое название), Регион, Количество ферм, Дата последней поставки.

- **Поиск** по названию — фильтрует список в реальном времени
- **Пагинация** — 25 записей на странице
- **Переход в карточку** — клик по строке открывает детальную страницу предприятия

### Карточка предприятия

Три вкладки:

**Сводка:**
- Реквизиты: полное название, ИНН, ОГРН, регион, юридический адрес, руководитель, телефон, email
- Два мини-графика динамики: соматические клетки по годам и жир/белок по годам

**Динамика качества:**
- График КМАФАнМ по годам (с линией норматива)
- График БГКП по годам
- График спор клостридий по годам
- Стековая диаграмма сортности (% Экстра / Спец. I / Спец. II по годам)

**Фермы и аудиты:**
- Таблица ферм: название, поголовье, дойное стадо, система доения, тип содержания
- История аудитов: дата, результат (с цветовым бейджем), срок одобрения, дата следующего аудита, просрочка

---

## 3. Страница «Аудиты»

Журнал аудитов хозяйств с акцентом на контроль сроков.

### Индикаторы в шапке

- 🔴 **Просрочено: N** — количество аудитов с истёкшим сроком
- 🟡 **Ближайшие 30 дней: N** — требующие внимания

### Фильтры

- Кнопка «Только просроченные» — показывает аудиты с нарушенным сроком
- Выбор результата: Одобрено / Условно / Отказано
- Счётчик итоговых записей

### Таблица аудитов

Колонки: Предприятие и ферма, Дата аудита, Результат, Одобрен на (мес.), Следующий аудит, Уведомление, Статус.

**Статус** отображается цветным бейджем:
- 🔴 `+N дн. просрочка` — на красном фоне
- 🟡 `Через N дн.` — ближайшие 30 дней
- 🟢 `✓ через N дн.` — всё в норме

Клик по названию предприятия — переход в карточку предприятия.

---

## 4. Страница «Аналитика»

Сводные графики по всем предприятиям.

### Фильтр по году

Выпадающий список для выбора конкретного года или «Все годы».

### Графики

- **Сортность по предприятиям** — горизонтальная стековая диаграмма: % Экстра / Сорт I / Сорт II по каждому предприятию
- **Соматика по предприятиям** — горизонтальный барчарт SCC с линией норматива (200 тыс/мл)
- **Целевые значения** — справочная таблица с нормативами по всем показателям

---

## 5. Страница «Нормативы»

Управление системой сортировки молока. Все изменения применяются немедленно — сорта пересчитываются автоматически при следующем запросе к API.

### Блок «Сорта молока»

Список активных сортов в порядке приоритета (от наивысшего к низшему). Логика: при расчёте сорта партии проверяется первый сорт, затем второй и т.д. — присваивается первый, которому соответствуют все показатели.

**Для каждого сорта:**
- Порядковый номер приоритета
- Цветная метка
- Название (редактируется кнопкой **✎ Переименовать**)
- Выбор цвета через color picker
- Кнопка **⏸ Выкл.** — исключить сорт из расчёта (партии этого сорта перестанут ему соответствовать)
- Кнопка **✕** — удалить сорт (с подтверждением)
- Внутренний код сорта (E / I / II / G1...) — показывается в интерфейсе

**Кнопка «+ Добавить сорт»:**
- Ввод названия нового сорта
- Автоматически создаётся внутренний код (G1, G2...)
- Автоматически копируется структура нормативных показателей из существующего сорта
- Новый сорт появляется в таблице нормативов

### Таблица нормативов

Сетка: строки = показатели качества, колонки = активные сорта.

**Показатели в таблице:**
Соматика, КМАФАнМ, Споры клостридий, БГКП, Плотность, Жир %, СЖК, Точка замерзания, Органолептика, Белок %, Температура, СОМО, Кислотность

**Редактирование:**
- Каждая ячейка содержит два поля: «от» и «до»
- Поле с изменёнными значениями выделяется синей рамкой
- Сохранение: нажать **✓** или Enter
- Пустое поле = отсутствие ограничения по этому параметру

**Кнопка «↺ Сбросить к ГОСТ 31449-2013»:**
Восстанавливает все граничные значения к стандартным (с подтверждением).

## 6. Импорт данных

Данные загружаются через скрипт командной строки из Excel-файлов формата Quality Monitor:

```
python scripts/import_xlsx.py --file "файл.xlsx" --enterprise "Название" --short "Краткое"
```

- Каждая строка листа `Quality` (начиная с 35-й) = одна партия
- При повторном импорте с флагом `--clear` старые данные удаляются
- Импортируются все 22 показателя качества

## Расчёт сорта молока

Сорт каждой партии рассчитывается автоматически в PostgreSQL функцией `calculate_grade()`:

1. Если обнаружены антибиотики → «Вне специфики»
2. Проверяются все активные сорта в порядке приоритета
3. Для каждого сорта проверяются все настроенные нормативы из таблицы `grade_standards`
4. Присваивается первый сорт, которому соответствуют все измеренные показатели
5. Если ни один не подошёл → «Вне специфики»

Значения NULL (показатель не измерялся) не влияют на расчёт — показатель считается прошедшим проверку.

| СОМО, % | ≥ 8.20 | ≥ 8.20 | — |
| Плотность, кг/м³ | ≥ 1027 | ≥ 1027 | — |

| Клостридии, НВЧ/л | ≤ 1000 | ≤ 3500 | ≤ 5000 |
